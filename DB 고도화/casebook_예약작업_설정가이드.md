# Casebook 사례 수집 예약 작업 — 설정 가이드

## 사전 준비 (한 번만)

1. 노션에 "Casebook 수집 기준 문서" 페이지를 만들고, 기준 문서 v1.0 내용을 붙여넣기
2. Cowork에서 노션 커넥터가 연결되어 있는지 확인
3. 아래 프롬프트의 [기준 문서 페이지명] 부분을 실제 노션 페이지 이름으로 교체

---

## 예약 작업 생성 프롬프트 (Cowork에 그대로 붙여넣기)

```
매주 월요일 오전 10시에 실행되는 예약 작업을 만들어줘. 작업 이름은
"Casebook 사례 후보 수집"이고, 작업 내용은 아래와 같아.

---

너는 Casebook 사례 DB의 수집 담당 리서처다. 아래 순서로만 작업한다.

1. 노션에서 "[기준 문서 페이지명]" 페이지를 먼저 읽는다.
   이 문서의 기준과 충돌하는 판단은 하지 않는다.

2. 기준 문서 7장의 "정기 확인" 소스 목록을 순서대로 확인해서,
   지난 실행일(7일 전) 이후 발행된 신규 글이 있는지 찾는다.
   검색 시 기준 문서 7장의 검색어 규칙을 따른다.
   ("온보딩" 단독 검색 금지 등)

3. 발견한 글을 기준 문서 2장(수록 필수 조건), 3장(즉시 제외),
   4장(날짜 규칙), 5장(신뢰도 등급)에 따라 판정한다.

4. 통과한 후보를 기준 문서 6장의 스키마로 정리한다.
   각 후보마다 다음을 포함한다:
   - 스키마 13칸 전체 (상태는 전부 "검토 대기")
   - 판정 사유 요약 (왜 통과인지, 애매한 부분은 무엇인지)
   - 신뢰도 등급과 그 근거

5. 절대 노션 DB에 직접 등록하지 않는다. 결과는 보고서 형태로만
   정리해서 나에게 전달한다. 후보가 0건이면 "이번 주 신규 후보
   없음"과 확인한 소스 목록만 보고한다.

6. 보고서 마지막에 "다음 중 수록할 항목 번호를 알려주세요"라고
   묻는다.
```

---

## 승인 후 등록 흐름 (예약 작업 실행 후)

보고서를 받으면 검토 후 이렇게 답하면 됩니다:

```
1번, 3번 수록해줘. 2번은 반려.
```

그러면 그 세션의 Claude가 노션 커넥터로 승인된 항목을 사례 DB에
"수록" 상태로 등록합니다. (반려된 항목은 등록하지 않음)

---

## 서비스 반영 (2주 주기 권장, 수동)

노션에 새 사례가 5~10건 쌓이면:

1. 노션 사례 DB → 내보내기 → CSV
2. antigravity(또는 클로드코드)에 CSV를 첨부하고 요청:

```
첨부한 CSV는 사례 DB 최신본이야. 기존 JSON 데이터 파일을 이 내용으로
업데이트해줘. 새로 추가된 사례가 몇 건인지 알려주고, 카테고리 값이
기존과 다른 새로운 값이 있으면 필터에도 반영해줘. 완료되면 빌드해서
에러 없는지 확인하고, Git 커밋 후 Vercel 재배포까지 진행해줘.
```

3. 배포 완료 후 실제 사이트에서 새 사례가 검색되는지 확인

## 운영 팁

- 첫 1~2회 실행은 결과를 꼼꼼히 검토하세요. 예약 작업이 기준을
  잘못 해석하는 부분이 발견되면, 대화로 고치지 말고 노션의 기준
  문서를 수정하세요 (기준 문서 9장 원칙).
- 후보가 매주 0건이어도 정상입니다. 기업 블로그의 좋은 회고 글은
  월 1~2건 나오면 많은 편이에요.
- 컴퓨터를 잘 안 켜두는 편이면, 로컬 예약 작업 대신 클라우드 예약
  작업(claude.ai/code/scheduled)으로 만들면 컴퓨터가 꺼져 있어도
  실행됩니다. 노션 커넥터는 양쪽 다 사용 가능합니다.
